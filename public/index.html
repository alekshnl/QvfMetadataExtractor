QVF Structure Analysis

Upload a QVF file and download its extracted metadata package.

This service reads the QVF file directly, decodes structured metadata blocks where possible, extracts the load script as a .qvs file, and returns the result as a ZIP archive.

Supported format: .qvf

Ready for upload

What happens to your app?

  1. The uploaded .qvf file is written to a temporary folder on the server.
  2. A Python extractor scans the QVF structure for metadata records, scripts, variables, data sources, and embedded assets.
  3. Structured findings are written into JSON files, the load script is saved as script.qvs, and raw evidence files are added for traceability.
  4. The extracted output is packaged into a ZIP file and sent back to your browser as a download.
  5. After processing, the uploaded file and temporary extraction folder are deleted from the server.

Technology used

  • Node.jsApplication runtime for the web service
  • ExpressHTTP server and static file delivery
  • MulterTemporary file upload handling
  • Python 3Extractor runtime for QVF analysis and ZIP generation
  • zlib + JSONDecoding of structured payload blocks found inside the QVF
  • zipfilePackaging of the metadata result for download
  • No Docker runtimeThe extraction path runs directly on the host without Qlik Core
