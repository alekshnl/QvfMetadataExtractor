Qlik Metadata Extraction

Upload a QVF file and download its extracted metadata.

This service imports a Qlik app into a local Qlik Core engine, runs the Qlik CLI unbuild process, and returns the extracted metadata as a ZIP file.

Supported format: .qvf

Ready for upload

What happens to your app?

  1. The uploaded .qvf file is written to a temporary folder on the server.
  2. The file is imported into a local Qlik Core engine that is only reachable from this server.
  3. The Qlik CLI runs the app unbuild process to extract the app metadata into files.
  4. The extracted output is packaged into a ZIP file and sent back to your browser as a download.
  5. After processing, the uploaded app file, temporary metadata files, and temporary engine app are deleted.

Technology used

  • Node.jsApplication runtime for the web service
  • ExpressHTTP server and static file delivery
  • MulterTemporary file upload handling
  • ArchiverZIP creation for the metadata download
  • Qlik CLIImport, unbuild, and cleanup commands
  • Qlik Core EngineLocal engine runtime used for processing the uploaded app
  • DockerServer-side runtime for the Qlik Core engine only
